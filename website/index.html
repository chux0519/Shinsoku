Product

One workflow model, multiple native entry points.

Shinsoku is not a single speech demo with a microphone icon on top. It is a workflow system: profile selection, provider routing, optional transforms, draft review, insertion, and history.

Profiles, not one-off toggles

Switch between dictation, review-before-insert, chat-style commit, and translation presets without rebuilding your setup every time.

ASR and post-processing can be split

Use one provider for recognition and another for cleanup or transform. Streaming and batch backends stay interchangeable.

Drafts stay in the loop

When recognition is close but not perfect, Shinsoku keeps a draft path instead of forcing you to abandon the workflow.

Native UI where it matters

Desktop remains Qt. Android stays Kotlin. iOS stays SwiftUI plus a keyboard extension. Shared logic lives below the shell.

Platforms

Built for the actual surfaces people type into.

Desktop

Full app workflow

  • Qt app for Windows, macOS, and Linux
  • Streaming ASR, history, HUD, clipboard and selection flows
  • Provider-assisted transforms and profile switching
Android

IME-first voice input

  • Native keyboard surface and settings app
  • Streaming or batch recognition with draft review paths
  • Shared native core underneath the Kotlin shell
iOS

App plus keyboard extension

  • SwiftUI app for dictation, drafts, and setup guidance
  • Keyboard extension for insertion and draft handoff
  • App Group-backed shared storage and shared mobile logic
Shared Core

Native core where reuse actually pays off.

The mobile stack is converging on a shared native core instead of letting Android and iOS drift forever. The shell stays native; the workflow rules do not.

Already shared

  • Transcript cleanup and commit planning
  • Built-in profile defaults and matching
  • Behavior summaries and transform summaries
  • Transform prompt generation
  • Post-processing runtime derivation
  • Provider diagnostics and provider display metadata

Design principle

Reuse the logic that is stable, expensive to re-implement, and likely to drift: prompts, profiles, routing, transform rules, summaries, diagnostics. Keep the surfaces native where they need to feel right.

Build

Everything stays in one repo without turning into a mess.

Desktop
cmake -S . -B build -G "Ninja Multi-Config" \
  -DCMAKE_TOOLCHAIN_FILE=/path/to/vcpkg/scripts/buildsystems/vcpkg.cmake
cmake --build build --config Release --target shinsoku
Android
cd mobile/android
./gradlew assembleDebug
adb install -r app/build/outputs/apk/debug/app-debug.apk
iOS
cd mobile/ios
xcodegen generate
xcodebuild -project ShinsokuMobile.xcodeproj \
  -scheme ShinsokuMobile \
  -destination 'platform=iOS Simulator,name=iPhone 16,OS=18.1' build

Shinsoku is a cross-platform voice input product with native shells and a shared workflow core.
