Product

One workflow model, multiple native shells

Profiles

Switch among dictation, review-before-insert, bilingual transform, and custom prompt flows without rebuilding your setup every time.

Backends

Use batch or streaming transcription, then run optional post-processing with a separate provider and model.

Native UI

Desktop stays in Qt. Android stays native. iOS is being built as SwiftUI plus a keyboard extension instead of forcing one UI toolkit everywhere.

Build

Use the repo without fighting its structure

Desktop
cmake -S . -B build -G "Ninja Multi-Config" \
  -DCMAKE_TOOLCHAIN_FILE=/path/to/vcpkg/scripts/buildsystems/vcpkg.cmake
cmake --build build --config Release --target shinsoku
Android
cd mobile/android
./gradlew assembleDebug
adb install -r app/build/outputs/apk/debug/app-debug.apk
